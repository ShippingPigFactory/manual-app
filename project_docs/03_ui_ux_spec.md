# 03. UI/UX詳細仕様書

## デザイン原則
- **SEO不要:** 社内ツールのため、メタタグやSSR最適化よりもレスポンス速度を重視。
- **スタイル:** SCSS Modulesを使用。クラス名の衝突を避ける。
- **トーン＆マナー:** 清潔感のあるオフィスライクなデザイン（青、グレー、白を基調）。ゲーミフィケーション（紙吹雪等）は禁止。

## 画面構成と仕様

### A. 全体レイアウト (Layout)
- **Header:** 常に上部に固定表示。
  - 左: アプリロゴ
  - 中央: **グローバル検索バー** (どの画面からでも検索可能にする)
  - 右: [管理者モードへ] リンク

### B. ダッシュボード (トップページ: `/`)
1.  **ヒーローセクション**
    - 画面中央に大きな検索入力欄を配置。
    - **Recommended Tags (おすすめタグ):** 検索窓の下に「よく使われる検索ワード」をチップ形式で並べる。
      - 例: `[入社手続き]` `[Wifi接続]` `[Slack設定]`
      - クリックすると検索結果画面へ遷移、または即座にフィルタリング。
2.  **マニュアル一覧エリア**
    - グリッドレイアウトでカードを表示。
    - **進捗表示:** 「習得度」ではなく「業務進行度」としてプログレスバーを表示 (例: `3/10 steps` 30%)。
    - ステータスバッジ: [未着手] [進行中] [完了]。

### C. マニュアル実行ウィザード (ページ: `/manual/[id]`)
- **レイアウト:** 集中モード。サイドバーなし、中央カラムレイアウト。
- **メインコンテンツ:**
  - `Step.title` を見出しとして大きく表示。
  - `Step.media` がある場合はメインビジュアルとして表示（画像クリックで拡大モーダル）。
  - `Step.content` を読みやすいタイポグラフィで表示。
  - `Step.checklist` をチェックボックスリストとして表示。
- **フッターナビゲーション:**
  - [戻る] ボタン: 最初のステップでは非表示。
  - [次へ] ボタン: 
    - `checklist` 内の `isRequired` 項目が全て `true` になるまで **Disabled** (無効化) 状態にする。
    - 無効時にクリックした場合、「未完了のチェック項目があります」とトーストまたはツールチップを表示。

### D. 管理画面 (ページ: `/admin`)
- **マニュアル作成・編集:**
  - 左カラム: ステップ構成ツリー（ドラッグ＆ドロップで並べ替え可能）。
  - 右カラム: 選択したステップの編集フォーム。
  - **キーワード設定:** マニュアル基本情報の編集画面に、検索用キーワード(`keywords`)の入力欄（カンマ区切り入力）を設置する。
