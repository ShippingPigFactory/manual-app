# 00. プロジェクト概要・要件定義

## プロジェクト名
**Internal Onboarding App** (社内業務習得システム)

## 目的
既存の「読みにくいマニュアル」を撤廃し、画面に従って操作するだけで業務が完了・習得できる「オンボーディング型アプリケーション」を開発する。
**社内ネットワーク（イントラネット）での利用を前提とし、SEO対策は一切考慮しない。**

## コアコンセプト
1.  **Reading < Doing:** 長文を読ませず、1つの画面で1つのアクションを実行させる。
2.  **Professional & Efficient:** ゲーミフィケーション（クエスト、レベル上げ）要素は排除。あくまで「業務進捗」を可視化するツールとする。
3.  **Search First:** 業務中に発生する「これどうやるんだっけ？」を即座に解決できる検索性を最優先する。

## ターゲットユーザー
1.  **一般ユーザー:** 業務手順を知りたい社員。ITリテラシーに依存せず操作できること。
2.  **管理者:** マニュアルを作成・更新し、社員の進捗を管理するリーダー層。

## 非機能要件
- **SEO:** 不要（`noindex` 設定、メタタグ最適化不要）。
- **パフォーマンス:** 社内ツールとしてのレスポンス速度を重視。
- **デザイン:** SCSSを使用し、清潔感のある実務的なデザイン。アニメーションは「操作フィードバック」のみとし、過剰な演出（紙吹雪等）は禁止。
